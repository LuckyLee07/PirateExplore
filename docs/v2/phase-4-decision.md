# V2 阶段 4：立项决策与下一阶段预算

日期：2026-07-16
当前决策：**HOLD（暂缓扩建，等待外部证据）**

## 1. 决策结论

V2 第一章已经具备可运行、可回归、可观察的正式样片候选，但尚未完成两轮外部目标用户测试和真实设备体验。因此现在不能给出 Go，也不能因为缺少外测而否定玩法方向。正确动作是冻结扩建、执行外测、修复 P0/P1，再按既定阈值复判。

以下情况才转为 **GO — 限额制作前三海域**：

- 第二轮独立首次远航率 ≥ 70%；
- 第二轮双阶段战斗理解率 ≥ 70%；
- 第二轮第二次远航意愿 ≥ 60%；
- 第二轮核心幻想首先回忆率 ≥ 70%；
- P0/P1 清零，内部质量门槛通过；
- 第二张地图经内容评审确认不超过 8 人周。

任一外测门槛未通过，决策保持 HOLD，安排 2～4 周纵切修正和新一轮验证；不直接制作十六海域。

## 2. 现有证据

已经具备：完整首章双路线闭环、双阶段战斗因果传递、失败恢复、返航升级、下一航程目标、正式资源映射、确定性数据导出、自动回归、本地行为记录和问题分级流程。

尚未具备：目标用户独立理解数据、第二次远航真实意愿、真实设备触控/音量/帧率记录。缺失项均明确保持 pending，不使用开发者体验或伪造模板行填充。

在复用现有数据源、状态机、表现容器、音频 bridge、QA profile 和验证脚本的前提下，第二张地图的上限预算为 7.5 人周：

| --- | ---: | --- |

| 6～8 个事件和对白 | 1.0 | 双路线差异、汇合点、角色反应完整 |
| 第二场敌船与战斗变体 | 1.0 | 复用双阶段规则并增加一个可解释变化 |

## 4. 前三海域限额

首章已完成纵切。立项通过后，建议只批准两张新增地图和发行基础，不一次批准十六海域：

- 第二张地图：7.5 人周；
- 第三张地图：6.5 人周，复用第二张形成的事件/敌人流程；
- 跨海域进度、内容选择与最终平衡：2.0 人周；
- 设备兼容、可访问性、商店页和发布准备：4.0 人周；
- 风险缓冲：2.0 人周。

总上限 22 人周。每张地图结束都运行完整质量门槛；若制作量连续两次超过估算 20%，暂停扩建并复盘工具链和内容范围。

## 5. 商业模式建议

建议采用“第一章完整免费试玩 + 一次性解锁基础航程”，后续只销售明确的新海域内容扩展。首章内不展示钻石商城、抽卡、付费复活、任务加速、随机宝箱续开或广告；失败恢复只使用游戏内可解释资源。

理由：

- 20 分钟首章已经能展示海图、船战、接舷、符文和升级，适合作为真实产品判断而不是付费诱饵；
- 一次解锁与以探索、决策为核心的体验一致，避免旧版强付费入口再次遮蔽幻想；
- 内容扩展的价值可以直接对应新海域、敌船、船员和符文，不破坏已有进度；
- 在第二次远航意愿尚未验证前，不应先建设复杂经济、商城或长期运营系统。

价格、区域和平台条款需在外测通过后另做市场验证，本阶段不拍脑袋给出定价。

## 6. Go 后的执行顺序

先做第二张地图的数据与灰盒，验证现有流程能稳定复制；再做第二场战斗变体和事件表现；通过中间质量门后才开始第三张地图。港口空间化、长期成长、成就、挑战、商店和引擎迁移保持 P2，不进入首个限额包。

最终决策表必须附上两轮原始匿名记录、指标计算、问题清零证据、真机检查和内容预算评审。缺一项则保持 HOLD。
