## 4. 全面屏与旧系统隔离

首次画面验证发现两个问题：

1. 旧任务管理器启动时弹出大号任务 Toast，覆盖 V2 样片；
2. 旧包没有现代 Launch Screen 声明，在全面屏设备上出现上下信箱黑边。

修复：

- 新增 `legacy.missions` 功能门，V2 不再初始化旧任务管理器；
- 在 iOS `Info.plist` 中加入 `UILaunchScreen`；
- 卸载旧包清理系统缓存后重新安装验证。

结果：Toast 消失，画面从设备顶边到底边完整显示，目标、操作按钮和页脚均位于可见安全范围内。

## 5. 音效故障与修复验证

### 5.1 旧后端复现

用 `NEWPIRATE_V2_AUDIO_CUE=cannon` 让 QA 构建在进入舰炮画面后调用旧 `AudioEngine.playEffect`。调用后应用退出到主屏，证明旧音频后端不能用于阶段 3。

### 5.2 新后端验证

修复后，Lua 通过 `playV2Sound` 绑定调用 AVFoundation：

- 资源从应用包 `assets/music` 解析；
- `AVAudioSession` 使用 Ambient 类别；
- `AVAudioPlayer` 负责短音效播放；
- V2 层不再包含 `AudioEngine.playEffect`。

重复相同的 `cannon` 自检，8.5 秒后应用仍稳定显示舰炮画面，未退出或卡死。静态校验同时确认六个 cue 文件均存在。

限制：自动化环境无法评价主观响度和音色；真机可听性、静音键与混音仍列入阶段 4 检查表。
